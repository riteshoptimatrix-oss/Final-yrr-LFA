================================================================================
                           1. LEAD SCORE LOGIC
================================================================================

The system calculates lead scores using multiple engines with weighted factors.

1.1 BASE LEAD SCORE CALCULATION (Scraper Level)
------------------------------------------------
Location: backend/src/source-core/base-source.ts

Algorithm:
- Start with base score: 30
- Add points for:
  - Has website: +20 points
  - Has phone: +10 points
  - Has email: +10 points
  - High rating (≥4.5): +10 points
  - Many reviews (≥50): +5 points
- Maximum score: 100

1.2 AI LEAD SCORE CALCULATION
-----------------------------
Location: backend/src/ai-sales-intelligence/lead-score-engine.ts

Algorithm (Weighted Scoring):
SEO (15% weight):
  - Score = (seoScore / 100) * 15

UI/UX (15% weight):
  - Score = (uiuxScore / 100) * 15

Responsive (15% weight):
  - Score = (responsiveScore / 100) * 15

Trust (15% weight):
  - Score = (trustScore / 100) * 15

Social Presence (10% weight):
  - Score = (socialPresenceScore / 100) * 10

Quality (10% weight):
  - Score = (websiteQualityScore / 100) * 10

Freshness (5% weight):
  - Fresh: +5 points
  - Moderate: +2.5 points

Contact Methods (5% weight):
  - Score = (contactMethods / 3) * 5

Activity (10% weight):
  - Activity Score = min(rating * 10, 5) + min(reviewsCount / 10, 5)
  - Final = (activityScore / 10) * 10

Final Score = Sum of all weighted components (capped at 0-100)

1.3 LEAD QUALIFICATION LEVEL
----------------------------
Based on AI Lead Score:
- High Potential: Score ≥ 85
- Medium Potential: Score 60-84
- Low Potential: Score < 60

                      3. RESPONSIVE/UX SCORE LOGIC
================================================================================

Location: backend/src/uiux-intelligence/responsive-score-engine.ts

3.1 RESPONSIVE SCORE (Max: 100 points)
--------------------------------------
Deductions:
  - Missing viewport meta: -20 points
  - Non-responsive layout: -25 points
  - Horizontal scroll (desktop or mobile): -20 points
  - Overflow issues: -10 points
  - Font size issues: -10 points
  - Off-screen elements > 5: -15 points (proportional)
  - Fixed width elements > 3: -10 points (proportional)

3.2 UI/UX SCORE (Max: 100 points)
---------------------------------
Critical issues: -15 points each
Warning issues: -8 points each
Info issues: -3 points each

Additional deductions:
  - Alignment issues: -5 points
  - Broken buttons: -10 points
  - Overlapping content: -10 points
  - Navigation issues: -8 points
  - Hidden content: -5 points
  - Spacing issues: -5 points

3.3 MOBILE EXPERIENCE SCORE (Max: 100 points)
---------------------------------------------
Deductions:
  - Missing viewport meta: -25 points
  - Not touch-friendly: -20 points
  - Font size issues: -15 points
  - Horizontal scroll (mobile): -20 points
  - Off-screen elements > 3: -15 points
  - Mobile layout broken: -20 points
  - Mobile-specific issues: -5 points each

3.4 SCORE LEVELS
----------------
- Good: ≥ 80 points
- Medium: 50-79 points
- Poor: < 50 points

================================================================================
                    6. WEBSITE QUALITY SCORE LOGIC
================================================================================

Location: backend/src/business-intelligence/website-quality-engine.ts

6.1 WEIGHTED QUALITY SCORE
--------------------------
Breakdown (all 0-100):
  - SEO: 20% weight
  - Responsiveness: 20% weight
  - UI/UX: 20% weight
  - Trust: 15% weight
  - Performance: 15% weight
  - Social Presence: 10% weight

Overall Score = (SEO * 0.20) + (Responsive * 0.20) + (UIUX * 0.20) +
                (Trust * 0.15) + (Performance * 0.15) + (Social * 0.10)

6.2 PERFORMANCE SCORE
---------------------
Response time < 1s: 100 points
Response time < 2s: 90 points
Response time < 3s: 75 points
Response time < 5s: 60 points
Response time < 10s: 40 points
Response time ≥ 10s: 20 points
Response time = 0 (failed): 0 points

                      7. LEAD QUALIFICATION LOGIC
================================================================================

Location: backend/src/services/website-analyzer.service.ts

7.1 WEBSITE STATUS DETERMINATION
--------------------------------
Based on issues count and quality score:

- No Website: Issues ≥ 3 OR responseTime = 0
- Broken Website: Issues ≥ 2 OR qualityScore < 30
- Outdated Website: qualityScore < 60 OR no modern structure OR no contact page
- Average Website: qualityScore < 80 OR no SSL
- Modern Website: All checks pass

7.2 QUALIFICATION LEVEL FROM SCORE
----------------------------------
- High Potential: Score ≥ 85 (website good, needs no improvement)
- Medium Potential: Score 60-84 (some improvements needed)
- Low Potential: Score < 60 (significant improvements needed)

Note: Higher scores = better quality websites (lower opportunity)
      Lower scores = worse quality websites (higher opportunity for services)

                  10. SCRAPING & DATA EXTRACTION LOGIC
================================================================================

Location: backend/src/services/scraper.service.ts
          backend/src/source-manager/

10.1 MULTI-SOURCE SCRAPING
--------------------------
Sources: Google Maps, JustDial, IndiaMart, Clutch

Process:
1. Build search query: "{businessType} in {area}, {city}, {state}"
2. Execute concurrent scrapes across all selected sources
3. De-duplicate based on company name + phone
4. Extract leads with metadata
5. Calculate initial lead scores
6. Store in database

10.2 DATA EXTRACTION FIELDS
---------------------------
- Company Name (required)
- Website
- Phone
- Email
- Address
- Category/Industry
- Rating (0-5)
- Reviews Count
- Source URL
- Source Metadata (JSON)

10.3 LEAD STORE PROCESS
-----------------------
1. Check for duplicate (companyName + phone)
2. If new: Create new lead document
3. If existing: Update with new data (merge strategy)
4. Trigger lead audit process (responsive + intelligence)
5. Return stored lead ID

================================================================================
                     11. AREA AUTOMATION LOGIC
================================================================================

Location: backend/src/automation/

11.1 SESSION INITIALIZATION
---------------------------
Engine: AreaAutomationEngine

Process:
1. Generate session UUID
2. Iterate cities → areas using AreaIterator
3. Create jobs for each (city, area, businessType) combination
4. Insert jobs into queue with position tracking
5. Start queue processing

11.2 JOB PROCESSING
-------------------
Queue: AreaAutomationQueue

Process per job:
1. Mark job as running
2. Execute scrape with sources, keyword (businessType), location
3. Update job with results (totalLeads, sourceResults)
4. Update session counters (totalLeads, completedJobs)
5. Mark job as completed
6. Process next job in queue

11.3 STOP MECHANISM
-------------------
1. Set stopRequested flag for session
2. Wait for current job to complete
3. Mark remaining pending/running jobs as skipped
4. Set session status to completed

11.4 RESUME MECHANISM
---------------------
1. Find skipped jobs for session
2. Reset skipped jobs to pending status
3. Reset session status to running
4. Re-enter same processing loop

================================================================================
                      12. CRM PIPELINE LOGIC
================================================================================

Location: backend/src/controllers/crm.controller.ts

12.1 PIPELINE STAGES
--------------------
1. new-lead → Initial lead entry
2. contacted → First contact attempt
3. interested → Shows interest in services
4. not-interested → Declines services
5. follow-up → Scheduled follow-up
6. meeting-scheduled → Meeting arranged
7. proposal-sent → Proposal delivered
8. negotiation → Discussion on terms
9. deal-won → Successfully closed
10. deal-lost → Lost opportunity

12.2 STAGE TRANSITIONS
----------------------
- Interest = "interested" → interested stage
- Interest = "not-interested" → not-interested stage
- Interest = "maybe-later" → follow-up stage
- Contact = "contacted" (no interest set) → contacted stage
- Follow-up date set (no contact/interest) → follow-up stage
- Proposal = "pending/sent" → proposal-sent stage
- Proposal = "approved" → negotiation stage
- Proposal = "rejected" → deal-lost stage
- Meeting = "scheduled" → meeting-scheduled stage

12.3 LEAD PRIORITY
------------------
Based on:
  - Lead score
  - Conversion probability
  - Revenue potential
  - Sales priority (AI)

----------------------------------------

META DESCRIPTION ANALYSIS (20 points possible):
-----------------------------------------------
- Length 120-160 characters: 20 points
- Length 1-119 characters: 10 points
- Length 161+ characters: 0 points (too long)
- No meta description: 0 points

HEADING STRUCTURE ANALYSIS (15 points possible):
------------------------------------------------
- Exactly 1 H1 tag: 15 points
- 2+ H1 tags: 10 points (multiple H1s is not ideal)
- No H1 tag: 0 points

IMAGE ALT TAGS ANALYSIS (15 points possible):
---------------------------------------------
- All images have alt attributes: 15 points
- Partial coverage: Proportional score (images with alt / total images * 15)
- No images: 0 points (no penalty for no images)

INTERNAL LINKS ANALYSIS (10 points possible):
---------------------------------------------
- 6+ internal links: 10 points
- 1-5 internal links: 5 points
- No internal links: 0 points

MOBILE FRIENDLINESS (10 points possible):
-----------------------------------------
- Has viewport meta tag: 5 points
- Has Bootstrap/Tailwind/Foundation: 5 points
- Has mobile web app meta tags: 5 points
- Any mobile indicator present: 10 points total
- No mobile indicators: 0 points

SSL/HTTPS ANALYSIS (10 points possible):
----------------------------------------
- HTTPS enabled: 10 points
- HTTP only (no SSL): 0 points

================================================================================
                    WEBSITE STATUS DETERMINATION
================================================================================

Website Status Logic:

1. NO WEBSITE
   - Response timeout (>15s)
   - Connection refused
   - DNS not found
   - Issues count >= 3

2. BROKEN WEBSITE
   - Quality score < 30
   - Issues count >= 2
   - Page doesn't load properly

3. OUTDATED WEBSITE
   - Quality score < 60
   - No contact page
   - No SSL (HTTP)
   - Old design patterns
   - Missing modern HTML5 elements

4. AVERAGE WEBSITE
   - Quality score < 80
   - Has SSL but not perfect
   - Has contact page but basic
   - Some outdated elements

5. MODERN WEBSITE
   - Quality score >= 80
   - SSL enabled
   - Contact page present
   - Modern HTML5 structure
   - Responsive design
   - Social links present
